# 品牌授权及验证平台 (Brand Authorization & Verification Platform) - 产品需求文档 (PRD)

## 1. 项目背景与目标
本项目旨在为品牌方提供一套规范化、数字化的经销商授权管理流程。通过自动化证书生成、多级审核流以及公开的真伪验证系统，建立品牌信誉，防止虚假授权，并简化经销商年检及证书下载流程。

## 2. 用户角色定义 (Role Definitions)

### 2.1 超级管理员 / 负责人 (Super Admin / Manager)
*   **权限等级：** 最高权限。
*   **核心功能：**
    *   **系统权限配置：** 添加、禁用审核员账号，重置经销商账号。
    *   **证书模板管理：** 上传证书底图（留空模板），设置字段填充坐标。
    *   **终审确权：** 对审核员初审通过的申请进行最后确认，点击签发后系统正式生成带章证书图。
    *   **全局预览：** 查看所有经销商的状态、授权历史及系统操作日志。

### 2.2 审核员 (Auditor)
*   **核心功能：**
    *   **申请初核：** 对经销商提交的新授权或续期申请进行资料核对。
    *   **流转状态：** 确认无误后点击“提交初审”，流转至负责人处进行终审。
    *   **退回/拒绝：** 发现资料问题可退回重填。

### 2.3 经销商 (Dealer)
*   **核心功能：**
    *   **首次登录强制改密：** 系统由管理员初始分配账号密码，经销商首次进入必须修改密码。
    *   **证书下载：** 在后台查看已生效的授权书，提供 PNG 高清图及 PDF 下载。
    *   **年检/更新申请：** 证书到期前可在线提交年检材料或更新授权时长申请。

### 2.4 访客 (Visitor/Consumer)
*   **核心功能：**
    *   **公开验证：** 在验证平台首页输入授权编号进行查询。
    *   **结果展示：** 系统验证编号真实性，若真实则展示授权主体、授权期限及防伪证书缩略图。

## 3. 核心功能流程 (Core Workflows)

### 3.1 证书核发流程 (Issuance Workflow)
1.  **资料录入：** 管理员或经销商提交授权申请（含名称、时长、授权范围）。
2.  **审核员确认：** 审核员核查数据准确性，点击“确认”。
3.  **负责人签发：** 负责人执行最后操作，系统自动将：
    *   文字（经销商名称、编号、有效期等）
    *   电子公章（透明底图）
    *   动态二维码
    *   自动渲染并覆盖到留空的证书模板上，生成正式文件。
4.  **激活：** 证书正式入库，经销商端可下载，访客端可查询。

### 3.2 访客查询流程
1.  访客访问平台域名。
2.  输入证书编号（唯一 ID）。
3.  系统匹配数据库。
4.  显示：【验证通过 / 已授权 / 证书合法】 或 【查询失败 / 未授权】。

## 4. 技术栈及设计语言 (Tech & Design)

### 4.1 技术选型
*   **前端框架：** Next.js (App Router) + Tailwind CSS / Vanilla CSS。
*   **后端服务：** Supabase (PostgreSQL, Auth, Storage)。
*   **图形引擎：** HTML5 Canvas / Sharp (用于自动化合成证书及电子章)。
*   **安全性：** Row Level Security (RLS) 确保经销商只能看到自己的数据。

### 4.2 设计标准 (NIHPLOD Brand Style)
*   **视觉风格：** 沿用 **nihplod.cn** 品牌调性。
*   **关键元素：** 
    *   **Neo-Lux (新奢感) 玻璃拟态：** 登录框、面板采用毛玻璃半透明背景。
    *   **Bento Grid (便签盒) 布局：** 管理后台面板采用整齐的卡片式错落布局。
    *   **微交互：** 审核状态流转时的平滑过渡动画，证书下载时的渐现反馈。

## 5. 开发建议路线图 (Development Roadmap)
1.  **Phase 1:** 初始化 Next.js 项目，配置 Supabase 数据库模型（经销商表、证书表、角色权限表）。
2.  **Phase 2:** 开发管理端仪表盘逻辑（两步审核流、模板配置功能）。
3.  **Phase 3:** 实现 Canvas 证书合成引擎。
4.  **Phase 4:** 经销商后台开发（首登改密、证书下载）。
5.  **Phase 5:** 访客验证页与 SEO 优化。

| :--- | :--- | :--- |
